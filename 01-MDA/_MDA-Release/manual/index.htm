I - Introdução

 

 

O projecto DigitArq propôs-se concretizar, entre outros objectivos, a conversão de instrumentos descritivos antigos que constituem uma parte significativa dos instrumentos de acesso à informação existentes no ADP. Este processo que abrangeu tanto documentos impressos como documentos digitais elaborados dento do Arquivo Distrital do Porto ou provenientes de entidades externas, resultou numa quantidade assinalável de material em formato electrónico constituído por ficheiros normalizados segundo as normas de descrição ISAD(G) e EAD. Tornou-se pois essencial armazenar os vários milhares de ficheiros resultantes do projecto num repositório de informação que permitisse não só instalar organizadamente esses ficheiros como também aceder à informação neles contida.

 

Uma ferramenta desta natureza -uma base de dados– permite facilmente não apenas o armazenamento e acesso a informação normalizada mas igualmente a produção de descrições de material de arquivo. Nesse sentido, julgou-se oportuno desenvolver um pequeno número de funcionalidades tendentes a facilitar e apoiar a produção normalizada de descrições arquivísticas.

 

Trata-se de um módulo que se articula com a pesquisa e recuperação de descrições realizada tanto local como remotamente (via web) e com o arquivo digital destinado à produção e gestão de objectos digitais que serão eventualmente associados às descrições armazenadas.

 

A estrutura de desenvolvimento desta ferramenta assentou em quatro normas internacionais:

  • ISAD(G) - International Standard Archival Description
  • EAD - Encoded Archival Description
  • ISAAR (cpf) - International Standard Archival Authorities Records (Corporate, Persons, Families)
  • EAC - Encoded Archival Context

 

As duas primeiras, harmonizadas desde a publicação da última versão do DTD do EAD em 2002, destinam-se a suportar a descrição e o processo de conversão. As duas últimas destinam-se a apoiar a produção de ficheiros de autoridade. Em ambos os casos são produzidos ficheiros codificados em XML de acordo com estruturas de texto normalizadas em EAD e EAC e portanto passíveis de troca e disseminação internacionais.

 

 

II – Requisitos mínimos de funcionamento e Instalação

O DigitArq existe em duas versões: Uma autónoma (standalone) em que a aplicação corre num computador individual. Outra de acordo com uma arquitectura de cliente servidor. No primeiro caso a aplicação instala um servidor local que funciona apenas no computador de instalação. No segundo caso é necessário haver um servidor ao qual acedam clientes e que aí seja configurado o SQL Server 2000 Standard edition.

b) Instalação do DigitArq

 

 

Introdução

 

 

O DigitArq é distribuído em duas versões distintas:

 

  • Standalone (autónomo)
    • Funciona localmente num computador sem precisar de rede ou servidor SQL. Pode ser utilizado apenas por um utilizador de cada vez.
  • Cliente/Servidor
    • Funciona como um cliente para uma base de dados central. Permite que diversas pessoas trabalhem em simultâneo sobre o mesmo conjunto de fundos.

 

 

Ambas as versões do DigitArq são distribuídas no mesmo CD de instalação. Ao colocar o CD no CD-ROM do computador o menu de instalação é automaticamente carregado (autorun).

 

 

 

 

Para que a instalação possa ser realizada com sucesso é necessário que o utilizador do computador tenha permissões de Administrador. Caso seja necessário deverá sair do Windows e reentrar com a conta de Administrador.

 

Cliente/Servidor

 

Para instalar a versão Cliente/Servidor do DigitArq basta seguir os passos descritos no programa de instalação, i.e., instalar pela ordem indicada os programas apresentados, bastando para isso seleccionar com rato o programa a instalar.

 

A ordem de instalação deverá ser a seguinte:

 

  1. Microsoft .Net Framework 1.1
  2. DigitArq 1.0
    1. Durante a instalação será perguntado se pretendemos que o DigitArq seja instalado apenas para o utilizador actual ou para todos os utilizadores. A esta pergunta devemos seleccionar a opção Todos os utilizadores (Everyone).

 

 

 

 

Standalone (autónomo)

 

Para instalar a versão standalone do DigitArq basta seguir os passos descritos no programa de instalação, i.e., instalar pela ordem indicada os programas apresentados, bastando para isso seleccionar com rato o programa a instalar.

 

A ordem de instalação deverá ser a seguinte:

 

  1. Microsoft .Net Framework 1.1
  2. Microsoft SQL Server Desktop Edition
  3. DigitArq 1.0
    1. Durante a instalação será perguntado se pretendemos que o DigitArq seja instalado apenas para o utilizador actual ou para todos os utilizadores. A esta pergunta devemos seleccionar a opção Todos os utilizadores (Everyone).

 

 

 

 

NOTA: Após terminada a instalação dos três programas, o computador terá que ser reiniciado para que as alterações façam efeito.

 

 

Servidor SQL

 

A versão Cliente/Servidor do DigitArq pressupõe a existência de uma rede TCP/IP (local ou Internet, embora a Internet nunca tenha sido testada) onde exista um servidor Microsoft SQL Server 2000.

 

NOTA: A instalação de um servidor SQL deverá ser efectuada por um técnico especializado em informática pois existem diversas configurações e parâmetros de segurança que devem ser ajustados com bastante precaução.

O DigitArq ao arrancar efectua uma série de testes ao sistema. Um desses testes consiste na detecção da base de dados que irá suportar toda a informação gerada pelo DigitArq. Após a instalação do SQL Server 2000 essa base de dados não existe, pelo que será criada pelo próprio DigitArq assim que este é executado.

2 Criar e Editar Registos

 

2.1 Criar registos

 

2.1.1 Após ter aberto um fundo, por baixo do menu principal, encontra na coluna da esquerda a árvore que representa a estrutura nivelar do conjunto documental a descrever. É aqui que irá surgir a estrutura do núcleo documental que se encontra a trabalhar à medida que a descrição for progredindo. Note que o nível de topo representa o fundo, a partir do qual poderá adicionar outros níveis inferiores, para isso bastará colocar o rato sobre o fundo (ou o nível hierárquico em que estiver a trabalhar) em que deseja criar registos, carregar no botão do lado direito do rato e seleccionar a opção Adicionar. Ao seleccionar esta opção, aparecem os níveis abaixo do nível em que se encontra, devendo então escolher o que lhe interessa.

 

2.1.2 Da mesma forma poderá escolher a opção Remover elemento, o que elimina o elemento que estiver seleccionado. Note que no caso desse elemento conter outros elementos ou ainda no caso de pretender eliminar mais que um elemento de cada vez, a aplicação lançará uma janela de interface perguntando se realmente deseja eliminar esses registos. Desta forma a possibilidade de eliminação acidental e involuntária é diminuída.

 

 

 

 

 

 

2.1.3 Do lado direito, tem disponível as zonas de informação descritiva de acordo com a estrutura da norma ISAD(G):

 

  • IDENTIFICAÇÃO,
  • CONTEXTO,
  • CONTEÚDO E ESTRUTURA,
  • ACESSO E UTILIZAÇÃO,
  • MATERIAIS ASSOCIADOS,
  • NOTAS,
  • CONTROLO DA DESCRIÇÃO.

 

2.1.4 De referir que neste sub-menu tem a possibilidade de visualizar todas as zonas de uma só vez, através da opção Mostrar tudo, ou então visualizar uma zona de cada vez, seleccionando a pretendida.

Se optar por esta forma, poderá trabalhar melhor, pois a lista não será tão extensa.

 

2.1.5 Os números entre parêntesis que aparecem à frente das zonas de informação, correspondem à quantidade de blocos que cada zona tem disponível para preencher. Este pormenor ajuda o operador, na medida em que, no caso de não haver campos a preencher, aparecerá um zero e nesse caso, não necessita de seleccionar essa zona.

 

2.1.6 Se quiser adicionar um novo registo debaixo da mesma série, ou de outro nível, poderá fazê-lo de três formas:

·       através da opção adicionar, que já referimos anteriormente;

·       premindo o botão replicar, que se encontra do lado direito ao fundo, o qual cria automaticamente um novo registo com a mesma informação do registo anterior; ou

·       premindo o botão novo, que se encontra do lado esquerdo do anterior, o qual cria um novo registo sem qualquer informação.

 

 

 

 

 

 

 

 

2.1.7 A opção Sair, tal como o nome indica, permite-lhe sair do programa.

 

 

 

 

 

 

 

 

2.2 Editar registos      

 

 

2.2.1 No Menu Editar tem disponíveis as opções: Cortar, Copiar , Colar, remover elementos, localizar e substituir.

Pode, através deste menu, deslocar os elementos dentro da árvore, para isso deve seleccionar os elementos que quer mover, seguido da opção cortar e depois colar onde quer o elemento. Pode igualmente arrastar com o rato o elemento para o local da árvore onde o deseja posicionar. Esta opção, no entanto, apenas permite mover um elemento de cada vez. Ao utilizar as funcionalidades de “copiar” ou “cortar”, pode seleccionar mais do que um elemento.

 

2.2.2 Quando se enganar a introduzir um elemento deverá utilizar a opção Cortar seguida de Colar se o quiser colocar num outro campo, se pretender eliminá-lo deverá usar a opção remover elemento. Esta funcionalidade elimina um ou mais registos simultaneamente. Sempre que pretender eliminar mais que um registo ou um registo que contenha debaixo registos-filhos, a aplicação solicitará confirmação para prosseguir.

 

NOTE BEM: Se apenas seleccionar um registo para remover, a aplicação NÃO solicita confirmação e elimina o registo. Esta eliminação é definitiva e irreversível.

 

 

 

 

 

 

2.3 Pesquisar registos

 

2.3.1 Para Localizar um elemento deverá seleccionar a opção referente e preencher os dados requisitados, na caixa de texto que aparece deve colocar o termo ou expressão que quer pesquisar.

O mesmo procedimento deverá ter quando necessitar de Substituir um elemento.

 

Note que apenas poderá efectuar a pesquisa no fundo em que se encontra a trabalhar e não noutro. Para aceder ao motor de pesquisa global que pesquisa em todos os fundos deverá seleccionar a opção ”pesquisa web” que se encontra no menu de “localizar e substituir”.

 

 

 

 

 

 

2.3.2 Na caixa superior escolhe o campo de descrição onde quer efectuar a pesquisa e na caixa inferior introduz o termo ou palavra que deseja ver pesquisado.

 

Existem também duas caixas de selecção que permitem escolher as seguintes opções:

 

2.3.3 Maiúsculas/minúsculas: Esta opção permite-lhe impôr como condição o termo a pesquisar estar obrigatoriamente escrito em maiúsculas ou minúsculas

 

2.3.4 Apenas palavras inteiras: Esta opção permite-lhe impôr como condição que apenas palavras inteiras, e não simplesmente conjuntos de caracteres, sejam seleccionados.

 

2.3.5 Expressão regular: As expressões regulares constituem um poderoso e flexível motor de pesquisa adequado essencialmente pesquisas textuais. Utiliza-se um conjunto de operadores, representados por símbolos significativos, que restringem, alteram ou ampliam os critérios de pesquisa. Os símbolos utilizáveis são os seguintes.

 

Símbolos de operadores:

 

 

 

  1. . - Satisfaz qualquer caracter
  2. [ ] - Classes de caracteres
  3. \w, \d - Atalhos para classes de caracteres
  4. ( ) - Agrupa tokens e cria retrospectos
  5. ? - Satisfaz ao token precedente zero ou uma vez
  6. + - Satisfaz ao token precedente uma ou mais vezes
  7. { } - Satisfaz ao token precedente um certo número de vezes
  8. ^ - Satisfaz ao conceito "início de uma linha" (largura-zero)
  9. $ - Satisfaz ao conceito "final de uma linha" (largura-zero)
  10. | - Satisfaz ao token à esquerda ou ao token à direita
  11. \ - Assume o caracter especial como literal (escape)
  12. \b - Limites de uma palavra

 

Operador 1: Satisfaz qualquer caracterè  .

O ponto (.) coincide com qualquer caracter.

Ex.  “..” irá retornar palavras que possuam apenas dois caracteres

 

 

Operador 2: Classes de caracteres è [ ]

Uma classe de caracteres coincidirá com um único caracter que estiver presente na classe.

 

Assim, "b[aei]d" coincidirá com "bad", "bed" ou "bid", mas não com "bod", "bzd" ou "b@d".

 

 

Operador 3: Atalhos para classes de caracteres è  \w, \d

Anotações resumidas (atalhos) estão disponíveis para algumas classes de caracteres. Pode usá-las individualmente ou como parte de uma classe de caracteres.

 

"\d" e "[\d]" são equivalentes a "[0-9]"

 

 

Operador 4: Associação è ()

Para agrupar uma série de tokens, basta pô-los entre parênteses: "(abc)+" coincidirá com abc, abcabc, abcabcabc, e assim em diante.

 

 

Operador 5: Opcional (zero ou um)è ?

Se um token é seguido de uma interrogação coincidirá ou uma ou nenhuma vez.

 

Por exemplo: "ab?c" coincidirá com "abc" e "ac".  Note que a interrogação é gananciosa, assim, abc terá precedência sobre ac.

 

 

Operador 6: Repetição (uma ou mais vezes)è +

 

Se um a token é seguido por um sinal de adição, aquele token será repetido uma ou mais vezes.

Exemplo: "ab+c" coincide com "abc", "abbc", "abbbc", e assim em diante

 

 

Operador 7: Repetição limitada (uma certo número de vezes) è{}

 

Use chavetas para indicar que um token deverá repetir-se um certo número de vezes.

  • "ab{5}c" coincidirá com "abbbbbc" e mais nada.
  • "ab{2,5}c" coincidirá com "abbc", "abbbc", "abbbbc" e "abbbbbc".
  • "ab{1,}c" é o mesmo que "ab+c".
  • "ab{3,}c" coincide com "abbbc", "abbbbc", e assim por diante, sem limite para o número de b's mas um mínimo de 3.
  • "ab{0,3}c" coincide com "ac", "abc", "abbc" e "abbbc".

 

 

Operador 8: Início de linha è^

 

O circunflexo coincide com o conceito de "início-de-linha".  Na verdade, ele não coincide com qualquer caracter.

 

Exemplo: "^ *[0-9]+" coincide com qualquer número no início da linha, precedido ou não de espaços

 

 

Operador 9: Fim de linha è $

 

O cifrão coincide com o conceito de "fim-de-linha".  Na verdade, ele não coincide com qualquer caracter.

A regex ".$" coincide com o último caracter de uma linha, qualquer que ele seja.

 

 

Operador 10: Ou è|

 

Coincide tanto com o token à esquerda ou à direita do pipe.

Este token tem a menor das precedências de todos os demais, assim você pode necessitar usar parênteses.

 

"ab|cd" coincidirá com "ab" e "cd".

 

"a(bb|cc)d" coincidirá com "abbd" e "accd".

 

Se dispuser de várias opções, deverá usar mais pipes: "a(bc|cd|qq)d" coincidirá com "abcd", "acdd" e "aqqd".

 

 

Operador 11: caracteres especiais è\

 

Se pretende usar quaisquer dos caracteres especiais:  . [ ] ( ) ? * + { } ^ $ | \ como literais nas suas expressões regulares, será necessário precedê-los com uma contra-barra (\).  Para coincidir uma palavra entre chaves, use "\{\w+\}".

 

Literais Especiais:

 

  • \n Nova-linha (LF, 0Ah)
  • \r Retorno-de-carro (CR, 0Dh)
  • \t Tabulação (09h)
  • \xXX Caracter, XX é o valor hexadecimal entre 00 e FF.

 

 

Operador 12: Limites da palavra è\b

 

\b coincide com os limites de uma palavra, isto é, o não-caracter entre um caracter de palavra (\w) e um não-caracter de palavra (\W).  Um caracter de palavra é aquele caracter que pode ser usado para formar palavras.  Isto não apenas inclui as letras de a-z, mas também letras acentuadas como é.  Um não-caracter de palavra é qualquer outro caracter que não seja um caracter de palavra.

 

\b pode ser usado para implementar o conceito "apenas palavras inteiras" numa pesquisa.  "\bcat\b" coincidirá com a palavra "cat", mas nunca com as três primeiras letras de "category".  Uma regex simples como "cat" coincidiria.

 

 

2.3.6 Outra forma de pesquisar um documento é através da opção Pesquisa web, em que abre o browser (Internet Explorer) com o interface de pesquisa geral e aí encontrará uma página com um formulário para colocar os dados das unidades de descrição que pretenda pesquisar.

 

 

 

2.4 Ver/Visualizar Registos

 

2.4.1 No Menu Ver temos as opções: Todos, Frequentes e Obrigatórios de acordo com os campos que pretendemos visualizar. Os obrigatórios são os que são assim considerados na norma ISAD(G) e na estrutura de descrição EAD. Os campos frequentes incluem os obrigatórios e são os que normalmente são preenchidos pelo operador. Finalmente a opção “todos” permite visualizar todos os campos de descrição que a base de dados suporta.

 

2.4.2 Dentro da opção Árvore podemos Expandir no caso de queremos visualizar todos os níveis de um fundo, e Colapsar no caso de queremos ver apenas o nível superior.

 

 

 

 

2.4.3 Poderá também ordenar os dados através da opção Ordenar Por, ordem cronológica, código de referência, título, nível de descrição ou ainda por data de criação. Esta última opção é especialmente útil quando pretender recodificar registos provenientes de uma incorporação. Escolhendo a opção de visualização por data de criação, os registos recém-chegados são colocados no fim dos que já lá se encontravam, facilitando assim a sua identificação e recodificação automática.

 

2.4.4 Se seleccionar a opção Confirmar gravação, sempre que alterar algum registo e ao sair do nível em que está, o programa vai-lhe pedir para gravar a alteração. Caso esta opção não esteja seleccionada, os registos são gravados automaticamente.

 

 

2.5 Importar e exportar registos

 

2.5.1 O Digitarq oferece várias possibilidades de importação e exportação de registos. Uma das  vantagens de se trabalhar com EAD é o facto de se poder criar ficheiros de descrições com estrutura normalizada, e assim compreensíveis por qualquer organização ou pessoa em qualquer parte do mundo.

Estas opções estão disponíveis a partir do menu “Fundos”

 

2.5.2 A Importação permita 3 opções:

 

1- EAD.

Neste caso é possível importar um ficheiro EAD para a base de dados. Aplica-se este caso, por exemplo, quando da recepção de ficheiros provenientes de incorporações elaborados a partir do módulo de gestão de incorporações (minidigitarq). Neste caso escolhe-se a opção EAD e selecciona-se o ficheiro que se pretende importar. A aplicação incluirá esses registos como fundo autónomo no caso de não haver qualquer referência idêntica na base de dados ou irá inseri-los dento do fundo com a mesma referência, posicionando-os no fim dos registos previamente existentes.

 

2- Fundo.

Esta opção destina-se a permitir incluir um fundo já existente na base de dados dento de outro fundo igualmente existente no repositório. Por vezes esta necessidade surge quando, no processo de descrição, se detecta que um determinado núcleo documental até aí considerado como fundo deverá ter uma relação de subordinação a outro fundo constituindo portanto um subfundo deste. Devemos escolher esta opção a partir do fundo no qual queremos inserir outro. Ao escolher esta opção surge a janela de gestão de fundos devendo escolher-se qual o que se pretende inserir.

 

3- Texto (separado por tabulações)

O DigitArq suporta a importação de ficheiros tipicamente designados por ficheiros de texto separado por tabulações. Este mecanismo de importação é útil em situações em que pretendemos integrar, num determinado fundo, registos oriundos de outras aplicações, tais como, Excel, FileMaker, Access, etc. Existem centenas de aplicações capazes de exportar os seus registos neste formato específico.

 

O ficheiro a importar deverá respeitar o seguinte formato:

 

  1. Cada linha do ficheiro representa um registo
  2. Cada coluna representa um campo do DigitArq
    1. São utilizadas tabulaturas para separar a diversas colunas
  3. A primeira linha do ficheiro deverá ser o cabeçalho
    1. O cabeçalho deverá ter o mesmo número de colunas que os registos subsequentes.
    2. Em cada uma das colunas deverá constar o nome campo do DigitArq a que essa coluna corresponde (ver em baixo os campos possíveis).
  4. Linhas em branco são ignoradas

 

 

Todos os registos importados são adicionados, de uma forma plana, debaixo do registo de Fundo. A arrumação dos registos nas correspondentes séries deverá ser levada a cabo por um operador.

3 Ferramentas de Apoio à Edição

 

 

No Menu Ferramentas temos a

 

3.1 Revisão automática que permite contabilizar e localizar os erros, como datas incorrectas ou ausentes, ausência de título, tipo de unidade de instalação ou referências incorrectas, esta revisão pode ser feita a toda a árvore ou apenas a um ramo específico da mesma. Sempre que escolhe esta funcionalidade aparece um aviso perguntando se deseja aplicar o processo a toda a árvore ou apenas ao ramo da mesma em que está posicionado. Caso pretenda que todo o conjunto de registos seja englobado no processo escolha “Sim”, caso pretenda que esse processo seja apenas aplicado à parte da árvore onde está posicionado, escolha “Não”.

Esta funcionalidade destina-se a dar maior flexibilidade à acessibilização de descrições aos utilizadores. No caso de conjuntos documentais muito extenso ou complexos em que a descrição total pode durar muito tempo, esta opção permite ir acessibilizando à consulta partes desse núcleo que vão sendo concluídas.

 

3.2 Inferência automática permite que a aplicação complete automaticamente determinados campos, como datas e extensão, inferindo a informação através dos campos respectivos preenchidos pelo operador nos níveis de descrição inferiores;

 

Note que a revisão e inferência são processos associados mas que decorrem autonomamente. Se o processo de revisão encontrar erros que não interferem com o processo de inferência - i.e., datas, e dimensões - a inferência decorre normalmente, sendo no entanto apresentado ao operador um relatório de erros detectados. Estes podem no entanto ser corrigidos mais tarde.

 

3.3 a Codificação automática serve para atribuir novos números de registos de forma automática. Esta opção é útil no caso de pretender recodificar um elevado número de elementos (por exemplo no caso de novas incorporações)

 

3.4 Por último a opção Estatísticas vai fornecer o número total de elementos registados naquele nível, discriminando os níveis de descrição presentes.

5 Administração

 

 

 

 

 

 

5.1 Este menu apenas está disponível para o administrador. Através das opções deste menu, ele poderá:

a) Criar ou remover os termos de indexação (Termos de indexação);

b) Criar, alterar, eliminar contas de utilizadores (Utilizadores)

c) Criar registos de autoridade (Registo de autoridade)

d) Validar ou invalidar os dados inseridos (Validação)

e) Tornar descrições visíveis ou invisíveis (Validação>Visível)

f) Aceder a estatísticas de desempenho pessoal e da aplicação (Registo de actividade, Estatísticas)

 

Vamos ver cada uma destas funcionalidades:

 

5.2 Criar ou remover os termos de indexação

Apenas os administradores podem criar e alterar termos de indexação. Isto porque à partida estes termos devem ser “controlados” para se tornarem mais eficazes e evitar repetições e redundância.

Para criar um termo novo o administrador deve escolher esta funcionalidade. Aparece então uma janela onde é necessário seleccionar a classe ou tipo de termo que vai inserir. Estas classes são neste momento as preconizadas na norma EAD (actividades, assuntos, famílias, funções, localidades, ocupações, organizações, pessoas) podendo no entanto criar-se as que se quiserem. Seguidamente cria o termo que deseja. Finalmente associa esse termo específico ao fundo no qual ele pode ser utilizado

 

 

 

 

5.3 Criar, alterar, eliminar contas de utilizadores

Ao seleccionar esta opção aparece uma janela com caixas para introduzir nome completo do utilizador, login e palavra-passe. Estas últimas são cifradas. Sempre que necessário poderá eliminar ou criar contas de utilizadores.

 

 

5.4 Criar/alterar registos de autoridade

Esta funcionalidade permite criar um registo de autoridade de acordo com a norma ISAAR(CPF) e EAC. É apenas aplicável aos níveis Fundo e Sub-fundo. Quando selecciona essa opção aparecem alguns campos automaticamente preenchidos e outros que terá de preencher manualmente.

Uma das opções possíveis é criar um ficheiro EAC que poderá ser exportado para repositórios de entradas de autoridade como por exemplo o LEAF da União Europeia.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

5.5 Validar ou invalidar os dados inseridos

 

Esta funcionalidade destina-se a validar as descrições. Por validar entende-se a observação das descrições produzidos e o reconhecimento que estas se encontram técnica e semanticamente correctas. Difere do processo de revisão na medida em que se trata do corolário de uma verificação manual por parte do administrador, enquanto o segundo é uma operação automatizada que detecta erros sintácticos e de estrutura.

 

Os processos de validar e tornar visível estão associados a sinaléticas e cores significativas. Antes de um registo ser validado possui a cor castanha e um asterisco (*) colocado à frente. A cor castanha significa que o registo não está visível ao público e o asterisco significa ausência de validação. Note que para um fundo estar visível tem primeiro que ser validado. Para o fazer selecciona o fundo ou partes do fundo que deseja validar e escolhe a opção “validação>válido”. Após esta operação estar concluída o asterisco desaparece.

Para tornar visível selecciona simplesmente a opção “visibilidade>torna visível”. Após esta estar concluída a cor muda para verde e os registos passam a ser visíveis aos utilizadores.

Note que pode aplicar estes dois processos ao fundo inteiro ou a qualquer parte do fundo. Imagine que se encontra a descrever um fundo grande e que completou a descrição de apenas uma série. Nesta situação tem a possibilidade de validar e tornar visível apenas essa série e não todo o fundo.

 

Sempre que a descrição validada seja por qualquer motivo alterada, esta deixa imediatamente de estar visível retornando ao estado de inválido. Torna-se então necessário recomeçar o processo.

 

Sempre que um fundo se encontra validado e visível as opções do menu alteram-se passando a oferecer a possibilidade de tornar inválido ou invisível.

 

 

 

 

 

 

 

 

 

 

 

5.6 Aceder a estatísticas de desempenho pessoal e da aplicação

 

A opção Registo de actividade permite consultar os registos de actividades de cada operador num determinado período de tempo à escolha. É também possível escolher as acções que se pretendem visualizar, ou seja, se o operador criou, alterou registos, acedeu à base de dados, etc. É ainda possível limpar o histórico eliminando todos os registos. Neste caso a aplicação guarda por defeito os dados referentes ao último mês..

 

 

O administrador tem também acesso às Estatísticas globais ou mensais dos registos introduzidos, sendo que as estatísticas globais se encontram agrupadas.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

ANEXO 1: Preenchimento dos Campos

 

 

 

 

(A cor vermelha indica campos de preenchimento obrigatório)

 

1 - Identificação

 

Nível de descrição – Posição de uma descrição da hierarquia de fundo.

 

Referência – Na referência deverão constar os códigos referentes ao País, serviço, nº de controlo ou identificador único. Pode ser alterada através da codificação automática. Este valor é sempre atribuído automaticamente pela aplicação, podendo no entanto ser alterado.

 

Título –  Inserir o título o qual pode ser próprio ou atribuído.

 

Datas extremas – Só são preenchidas nos níveis hierárquicos inferiores (ex.: unidade de instalação, documento), nos outros níveis superiores, as datas serão calculadas através da inferência de dados.

 

Datas predominantes – É o grupo de datas ou data única onde há maior incidência de documentação.

 

Dimensões – Indica-se a extensão da unidade de descrição. Este valor é inferido automaticamente pela aplicação

 

Resumo – Breve descrição do tema.

Tipologia – Tipo diplomático ou funcional do documento

 

Termos de indexação : Vocabulário controlado pelo Arquivo. Os fundos já dispõem de uma lista com estes termos, para ter acesso a essa lista bastará premir a seta verde que se encontra junto a este campo, caso queira criar um novo termo deverá comunicar ao administrador, pois só ele o poderá fazer.

 

Entidade detentora – A instituição que custodia os documentos.

 

 

 

 

 

 

 

 

 

 

2- Contexto

 

Autor/Produtor: O Autor é a pessoa singular ou instituição responsável pelo conteúdo intelectual do documento.

O produtor é a pessoa colectiva, família ou pessoa singular que produziu, acumulou e ou conservou documentos no decurso da actividade.

 

Morada – Morada do Autor/Produtor.

 

Destinatário – A quem se destina o documento.

 

Morada – Morada do destinatário.

 

Notário – Nome do notário que escreveu e assinou o documento.

 

Escrivão – Nome do escrivão que escreveu e assinou o documento.

 

História Administrativa/Biográfica – O objectivo é contextualizar os documentos de arquivo e torná-los mais compreensíveis, indicando a história administrativa e biográfica.

Se houver informação adicional publicada, citar a fonte.

No caso de serem pessoas singulares ou Famílias deverão constar: Nomes completos; Data de nascimento e de Morte; Moradas; Actividades; Realizações significativas; Local de Nascimento.

No caso de serem pessoas colectivas deverão constar: Nome oficial, Funções, Missão, Evolução, Hierarquia Administrativa. Nomes anteriores e posteriores, variantes do nome.

 

Localidade: Localidade da pessoa ou empresa  que detinha o documento.

 

Lista cronológica: Indicar datas e feitos da pessoa em questão por ordem ascendente.

 

História custodial e arquivística  - Faculta a informação sobre a História da unidade de descrição na sua autenticidade, integridade e interpretação.

Registar sempre:

- Transferências de Propriedades;

- Circunstâncias que tenham contribuído para a sua estrutura e organização actual (ex. História da organização, instrumentos de pesquisa, reutilização de documentos, migrações de Software).

Nota: Se a História Custodial for desconhecida registar a informação.

 

Modo de aquisição -  Tem como objectivo identificar a origem imediata de aquisição ou transferência da unidade de descrição, a partir de que unidade foi adquirida, data ou forma de aquisição

(se não forem confidenciais).

Pode acrescentar números ou códigos de entrada de documentos.

Nota: Se for desconhecida registar essa informação.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

3- Conteúdo e estrutura

 

 

Âmbito e conteúdo -  O conteúdo ou temática da unidade de descrição

 

Organização -  Deverá constar informação sobre a  estrutura interna, ordenação e sistema de classificação de unidades de descrição.

Deve informar que tipo de tratamento documental foi utilizado pelo arquivista. Para documentos electrónicos não esquecer de referenciar informação sobre a arquitectura do sistema.

 

Avaliação – Faculta a informação relativa a qualquer avaliação, selecção e eliminação.

Registar sempre: Acções de validação, selecções e eliminações ocorridas ou planeadas para a unidade de descrição.

 

Ingressos adicionais – Faculta a informação relativa a qualquer incorporação a ocorrer ou planeada para a unidade de instalação, informa sobre as entradas complementares previstas relativas à unidade de descrição. No caso de se aguardarem ingressos adicionais, dar uma estimativa da sua quantidade e frequência (ex. Recortes de revistas, publicações oferecidas, colecções).

 

Plano de classificação– Deverá ser incluído o plano de classificação utilizado para a unidade de descrição

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

4- Acesso e Utilização

 

Estatuto Legal – Estatuto legal do documento.

 

Condição de acesso – Informação sobre o estatuto legal ou outras disposições que restrinjam ou afectem o acesso à unidade de instalação.

Dever-se-à especificar a legislação ou estatuto local, acordos, regulamentação ou decisões que afectem o acesso à unidade de descrição.

Se houver algum período de restrição do acesso deve ser indicada, assim como a data a partir da qual se poderá consultar os documentos em causa.

 

Condições de reprodução – Deve-se indicar as restrições à reprodução da unidade de descrição se existirem, como poderá ser o caso de direitos de autor que regulam a produção da unidade de descrição. Se não houverem restrições não é necessário assinalar, se forem desconhecidas deve-se assinalar.

 

Idioma – Identificar o tipo de idioma, o tipo de alfabeto, escrita, sistema de símbolos ou abreviaturas utilizadas.

 

Características físicas – Faculta a informação sobre características físicas ou requisito técnico relevante que afectem a utilização da unidade de descrição nomeadamente por razões de preservação. Mencionar qualquer Hardware ou Software necessário para aceder à unidade de descrição.

 

Detalhes específicos dos materiais – Indicar aspectos particulares dos materiais a descrever, como se tratar de documentos electrónicos, indicando nesse caso o software e hardware utilizados para a sua produção; outro exemplo é a presença de iluminuras, gravuras, etc.

 

Aspecto físico – Informação complementar da anterior. Indica-se por exemplo aspectos diplomáticos internos e externos do documento: tipo de letra, tipo de suporte, etc.

 

Localização física – Onde se encontra fisicamente o documento. No caso de documentos electrónicos, indicar a localização do suporte (CD, banda, etc.)

 

 

 

 

 

 

Numeração original – Indica-se a cota ou nº original do documento

 

Instrumentos pesquisa – Informar sobre os instrumentos de pesquisa existentes, elaborados pela entidade detentora, ou pelo produtor.

Informar como obter uma cópia.

 

 

 

 

5- Materiais associados

 

Localização original – Indicar a existência, localização, disponibilidade e ou destruição dos originais, quando a unidade de descrição é uma cópia/reprodução.

Se o original estiver disponível indicar a localização, bem como quaisquer números de controlo significativos.

Se não existirem originais ou se não se souber a localização, indicar sempre.

 

Existência de cópias – Indica a existência, localização e disponibilidade das cópias da unidade de descrição, se houver cópias a registar a sua localização bem como os números de controlo significativos. Inserir aqui informação sobre existência de microfilmes ou imagens digitais.

 

Materiais relacionados – Registar informação sobre unidades de descrição existentes na mesma entidade detentora, ou em qualquer outro lugar, que sejam relacionadas com a unidade de descrição, pela proveniência ou outro tipo de associação.

 

Material separado – Todo aquele material que, embora não faça parte da unidade de descrição, possa complementar a sua informação.

 

Bibliografia – Referências bibliográficas inseridas de acordo com normas da série NP-405.

ANEXO 2: Ficheiros DigitArq

 

 

O DigitArq é composto por uma série de ficheiros anexos que residem na pasta onde o DigitArq foi instalado. Segue-se uma breve explicação da função de cada um desses ficheiros.

 

eac.xslt

Quando um registo de autoridade é exportado para EAC, a stylesheet EAC.XSLT é aplicada automaticamente ao documento exportado.

Esta stylesheet é útil na eventualidade da norma EAC (que ainda se encontrava em draft quando concluímos o projecto) ser modificada. O ficheiro disponibilizado pelo DigitArq não altera o documento EAC gerado pela aplicação, mas se quisermos que o resultado da exportação gere outro tipo de XML, basta substituir esta stylesheet por uma que efectue a transformação desejada.

 

ead2cnf.xslt

Stylesheet utilizada para converter documentos XML exportados em EAD para CNF (CALM Natural Format) para serem importados pelo DScribe CALM.

 

 

database.sql

Descrição SQL da base de dados do DigitArq. Ao arrancar o DigitArq verifica se o servidor de base de dados se encontra na rede. Caso este exista, mas a base de dados não esteja criada (e.g. primeira utilização da versão standalone) o digitarq utiliza as instruções contidas neste ficheiro para reconstruir a base de dados.

 

DigitArq.exe.config

Ficheiro de configuração do DigitArq. Toda a interface e mensagens apresentadas pelo DigitArq podem ser alteradas neste ficheiro XML.

Caso se pretenda traduzir o DigitArq para outro idioma, é este ficheiro que deve ser editado.

 

Secções configuráveis

 

mandatoryFields

Definição dos campos que deverão aparecer no DigitArq para preenchimento quando escolhemos a opção ver apenas campos obrigatórios.

 

frequentFields

Definição dos campos que deverão aparecer no DigitArq para preenchimento quando escolhemos a opção ver campos frequentes.

 

allFields

Definição dos campos que deverão aparecer no DigitArq para preenchimento quando escolhemos a opção ver todos os campos.

 

visualFieldsLabelSettings

Definição do texto apresentado nas etiquetas visuais do DigitArq normalmente associadas ao preenchimento de campos. Pode ser alterado caso de deseje traduzir o DigitArq para outro idioma.

 

sqlServerSettings

Definição dos parâmetros de acesso à base de dados. Esta secção apenas deve ser alterada caso existam fortes razões para o fazer e apenas por técnicos especializados. Um exemplo da necessidade de alteração destes parâmetros poderá ocorrer, por exemplo, quando o Servidor SQL mudar de endereço.

 

  • ServerAddress
    • Endereço do servidor SQL
  • Database
    • Nome da base de dados onde se encontram os registos do DigitArq.
  • Username
    • Nome do utilizador para acesso à base de dados.
  • Password
    • Password do utilizador
  • SA_Username
    • Nome do utilizador com privilégios de administração sobre a base de dados
  • SA_Password
    • Password do utilizador com privilégios de administração

 

NOTA: As definições do SA apenas são necessárias para criar uma base de dados quando esta ainda não existe no sistema. Caso esse não seja o caso, ou seja, a base de dados já exista e esteja devidamente configurada, estas definições deverão ser removidas por razões de segurança.

 

logMessages

Mensagens escritas na tabela de registo de actividade.

 

infoMessages

Mensagens informativas apresentadas pelo DigitArq. Caso se pretenda traduzir o DigitArq, esta secção também deverá ser traduzida.

 

Constants

Definição de algumas constantes úteis ao sistema.

 

  • AdvancedWebSearchURL
    • Endereço da página de pesquisa avançada na Web
  • GlobalStatsURL
    • Endereço da página de ferramentas – Estatísticas globais
  • MonthlyStatsURL
    • Endereço da página de ferramentas – Estatística mensais
  • LogViewerURL
    • Endereço da página de ferramentas – LogViewer

 

errMessages

Mensagens de erro apresentadas pelo DigitArq. Caso se pretenda traduzir o DigitArq, esta secção também deverá ser traduzida.

 

contextMenu

Definição dos itens que figuram nos menus de contexto que são apresentados quando carregamos com o botão direito do rato sobre um elemento da árvore no DigitArq.

 

appSettings

Definição de algumas etiquetas do DigitArq que não foram definidas na secção de VisualFieldsLabelSettings. Constam desta secção a definição do texto a figurar nos menus, títulos de janelas, etc.

Caso se pretenda traduzir o DigitArq, esta secção também deverá ser traduzida.
